## Context

The current `make all` flow runs `build2`, `libspikeso`, and `sim`. `build2` reads firmware inputs from `firmware/<TEST_NAME>/` but writes generated objects and images into `firmware/<TEST_NAME>/obj/`. `libspikeso` reads C++ sources from `src/cosim/` and writes `scripts/libspike.so` plus `scripts/libspike.vpi`. `sim` reads `src/top/testbench.v` and `external/picorv32/picorv32.v`, then writes `src/top/testbench.vvp`.

This mixes generated artifacts with project-owned source directories. The requested target is to place firmware-generated files under `build/firmware` and source-generated files under `build/src`.

## Goals

- Keep source/input directories free of active build outputs.
- Preserve `make all` behavior and active co-simulation semantics.
- Make generated paths configurable through Makefile variables.
- Keep `clean` focused on generated output directories.

## Non-Goals

- Do not reorganize source files under `firmware/`, `src/`, or `scripts/`.
- Do not modify firmware contents, linker scripts, RTL behavior, or Spike behavior.
- Do not fully repair legacy upstream PicoRV32 Makefile targets that are not part of the active `make all` path.

## Design

Introduce explicit source/build directory variables:

```text
BUILD_DIR          ?= $(TB_HOME)/build
FIRMWARE_DIR      ?= $(TB_HOME)/firmware
TEST_SRC_DIR      ?= $(FIRMWARE_DIR)/$(TEST_NAME)
FIRMWARE_BUILD_DIR ?= $(BUILD_DIR)/firmware
TEST_BUILD_DIR    ?= $(FIRMWARE_BUILD_DIR)/$(TEST_NAME)
SRC_BUILD_DIR     ?= $(BUILD_DIR)/src
COSIM_BUILD_DIR   ?= $(SRC_BUILD_DIR)/cosim
TOP_BUILD_DIR     ?= $(SRC_BUILD_DIR)/top
```

`TEST_SRC_DIR` remains the input location for `.S`, `.c`, headers, and linker scripts. `TEST_BUILD_DIR)/obj` becomes the output location for active firmware artifacts. `MY_ELF_PATH` and `MY_HEX_PATH` should point at the build directory so Spike, VPI, and Verilog all consume the same generated files.

`VPI_MODULE_DIR` should default to `$(COSIM_BUILD_DIR)` and `SIM_VVP` should default to `$(TOP_BUILD_DIR)/testbench.vvp`. The build recipes must create those directories before writing artifacts.

The Verilog testbench fallback paths should match the new default generated firmware paths for direct runs without plusargs. The Makefile still passes plusargs explicitly during `make all`; the fallback update prevents stale source-tree paths from being used in ad hoc runs.

Documentation should describe `build/` as the generated-output tree:

```text
build/
  firmware/<TEST_NAME>/obj/
  src/top/testbench.vvp
  src/cosim/libspike.so
  src/cosim/libspike.vpi
```

## Risks

- `TEST_DIR` currently means both input and output location. Splitting it into source/build variables is necessary to avoid looking for source files under `build/`.
- `vvp -M` must be updated consistently with the generated `libspike.vpi` location.
- Direct testbench execution without plusargs depends on hard-coded fallback strings in Verilog.
- Existing generated files in old locations may remain on disk until cleaned manually, but active builds should no longer produce them.

## Verification

- Run `git diff --check`.
- Run `make clean`.
- Run `make all` with the required modules loaded.
- Confirm the active outputs are generated under `build/firmware` and `build/src`.
- Confirm old active output locations are not regenerated by `make all`.
